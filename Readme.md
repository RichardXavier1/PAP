RESUMO DO PROJETO — PAP
Nome

    PAP — Plataforma de Acompanhamento e Psicoeducação

Propósito central

    Oferecer uma ferramenta de uso diário que ajude pessoas a:

    acompanhar seu estado emocional ao longo do tempo

    organizar informações pessoais relevantes

    acessar conteúdo psicológico confiável e responsável

    se preparar melhor para acompanhamento profissional

O PAP não diagnostica, não trata e não substitui psicólogos ou psiquiatras.

Objetivos do projeto

Para o usuário

    Autoconhecimento baseado em registro, não em achismo

    Redução de desinformação sobre saúde mental

    Melhor aproveitamento de terapia (quando houver)

    Orientação clara sobre quando buscar ajuda profissional

Estrutura do PAP (por camadas)

Camada 1 — Núcleo (MVP)

Funções essenciais:

    Cadastro e autenticação de usuários

    Diário emocional estruturado

    Registro de eventos importantes

    Histórico e visualização simples

O PAP já funciona só com isso.

Camada 2 — Psicoeducação

Funções:

    Conteúdos informativos revisados por estudantes/profissionais de psicologia

    Categorias (ansiedade, estresse, emoções, etc.)

    Linguagem acessível e avisos de responsabilidade

Informação confiável ≠ conselho clínico.

Camada 3 — Insights simples

Funções:

    Relações entre hábitos e humor

    Observações descritivas, não conclusivas

    Mensagens do tipo “foi observado que…”

Nada de rótulos ou diagnósticos.

Camada 4 — Preparação para terapia

Funções:

    Anotações para sessões

    Organização de temas recorrentes

    Exportação de dados (PDF ou texto)

Camada 5 — Encaminhamento e apoio

Funções:

    Orientação sobre sinais de alerta

    Contatos oficiais (CVV, CAPS, SUS, etc.)

    Instruções claras em situações de crise

Limites éticos (definidos desde o início)

O PAP NUNCA:

    faz diagnóstico psicológico

    aplica testes clínicos

    sugere tratamento

    substitui acompanhamento profissional

    usa linguagem de terapeuta

Esses limites:

    aparecem na interface

    aparecem nos textos

    aparecem na documentação

Colaboração com psicologia

consultar profissionais:

    revisam conteúdos

    validam linguagem

    ajudam a definir o que não pode ser dito

    não interferem na lógica do sistema

    Você é o responsável técnico.

Direção técnica (inicial)

    Aplicação Web com foco em uso mobile, arquitetura modular
    preparada para crescimento por camadas, backend web em Python,
    banco de dados simples no início e frontend responsivo com foco
    em clareza, acessibilidade e baixo atrito. Possível evolução
    futura para PWA e aplicação mobile.

Verdade final do resumo

O PAP:

não tenta “resolver saúde mental”

resolve organização, informação e acompanhamento

cresce por camadas

é viável, ético e útil

é um projeto sério, não acadêmico genérico
